1. semantic (시맨틱 요소) 
    : 시맨틱 태그를 만들어 브라우저에게 요소의 의미나 목적을 명확하게 알려주는 요소이다.
	(기본적인 웹의 태그들은 문서의 구조에 대한 정보는 주지 않기 때문.)
 
     -> 기존의 웹보다, 시맨틱 태그를 사용한 웹의 코드가 훨씬 간결하다.  

2. semantic tag (시맨틱 태그)
   
   <header> : 문서의 머리말.
   <hgroup> : <h1> ~ <h6> 요소들의 그룹.
   <nav> : 내비게이션 링크.
   <article> : 문서의 내용이나 블로그 포스트.
   <section> : 문서의 섹션.
   <aside> : 사이드바와 같이 옆에 위치하는 내용.
   <footer> : 문서의 꼬리말.
   <figure> : 그림이나 도표.
   <time> : 날짜와 시간을 표시.

3. 문서의 구조와 스타일 (시맨틱 태그와 CSS의 관계성)
   : 시맨틱 태그를 사용하여 HTML 문서를 작성하더라도 웹브라우저가 자동적으로 색칠을 
     하거나 위치를 잡아주지 않는다.
     위치를 잡아주고 스타일을 지정하는 것은 CSS의 영역이기 때문이다.
     시맨틱 태그를 사용하더라도 CSS가 없으면 아주 썰렁한 텍스트만이 출력될 뿐이다.
